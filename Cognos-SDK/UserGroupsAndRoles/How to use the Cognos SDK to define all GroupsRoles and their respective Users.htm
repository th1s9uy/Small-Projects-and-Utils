What does the code do?

How do you use the Cognos Software Development Kit (SDK) to define all Groups/Roles and their respective Users?

How does the code work?

This document provides a Software Development Kit (SDK) sample that queries the Content Store, retrieves all Groups and Roles, and then defines the respective Members. The output is then printed to a file.

Note that the sample will recursively define all users belonging to a particular Group or Role. This means that if one Role is a member of another, the sample will retrieve all Users that belong to the child Role as well. For example, if RoleA has RoleB and User1 as Members and RoleB has User2 as a member, when the script will define the Members as follows:

Role: RoleA
User1
User2

Role: RoleB
User2

Use the attached sample to retrieve all Groups/Roles and define their respective users.

How do you run the code?

1. Extract the attached sample.

2. Change the following lines in order for the sample to be compatible for your environment. For example, for ReportNet:

// Variable that contains the default URL for CRN Content Manager.
String reportNetEndPoint = "http://localhost/crn/cgi-bin/cognos.cgi";
// must be a user with System administrator privledges.
String userName = "administrator";
String password = "password";
String nameSpaceID = "SDK";
String fileName = "C:/Cognos Namespace Configuration.txt";

3. Compile and Run the sample as per the SDK Developer Guide.

Note: If you have a Series 7 namespace defined as an authentication source for Cognos ReportNet, this sample can also be used to define the users belonging to each User Class (Role) in your Series 7 namespace. This is done by simply changing the ":" in the line

String nsToQuery = ":";

to the namespace ID of the Series 7 namespace. ("SDK" for the above example)

For more information see the SDK documentation "Getting Started" to setup the toolkits.
